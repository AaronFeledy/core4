## Codebase Patterns

- Env-gated native acceptance preflights host, explicit opt-in, and tooling as structured skips; once prerequisites pass, command failures remain failed outcomes even when release policy makes the cell advisory.
- Windows managed Podman machines expose the API at `\\.\pipe\podman-lando`; the data-root `runtime/run/podman.sock` endpoint is the native managed-service path, not the win32 machine endpoint.
- Bun subprocess `FileSink.write()` completion values are not partial-write offsets. Send each bounded chunk once, await `write`/`flush`, and await `end` before treating stdin delivery as complete.
- MCP outbound queue caps do not bound pre-queue JSON serialization; enforce the same byte budget during traversal before retaining the completed JSON string.
- Built-in OCLIF wrappers must not parse before `LandoCommandBase.runEffect`; shared renderer-aware flag validation and parsing live in the base command.
- A clean-tree inventory gate may bootstrap only when its generated corpus is wholly empty; partial corpora must remain strict so missing-artifact diagnostics are not repaired away.
- Runtime-bundle publishers pin actions to full commit SHAs and resolve Ubuntu build packages through a fixed APT snapshot; bind each installed candidate's own policy stanza to that snapshot under `pipefail` without early-exit pipelines.
- Landofile authoring tools must enumerate `presentLandofileLayers` and route each form through the existing YAML/TypeScript loader seam; include refresh must gather authored layers without invoking pin-mode include resolution.
- Landofile lint validates each authored layer before validating the merged document; merge precedence must not hide a layer shape that runtime loading rejects.
- Podman imports host certificates for `machine set --import-native-ca` during machine startup; an already-running Lando-owned machine must restart after enabling the setting for trust sync to complete.
- Host-proxy transport diagnostics discover ownership-valid worker records under `makeLandoPaths().hostProxyRunRoot` and reuse the authenticated `probeWorker` control path; absent worker state is an intentional no-op, not a warning.
- Main-binary builds go through `scripts/build-compiled-binary.ts`: OpenTUI requires ESM bytecode, and multi-target release builds must install all locked optional native packages with frozen `--os=* --cpu=*` inputs before compiling.
- Programmatic `Bun.build` may resolve with `success: false`; compile wrappers must inspect the result, preserve diagnostics, and fail the calling release stage instead of trusting promise resolution.
- Promise-backed interactive drivers must accept the enclosing Effect cancellation signal and settle through their normal cleanup path so interruption destroys terminal renderers and listeners.
- Split-footer resize handling must recompute semantic footer content at the new dimensions before replay; eventless input changes such as full-tail paging require an explicit repaint.
- Expanded transcript resize repages through the serialized event/input seam at the new row budget while preserving the current file-tail anchor.
- Build orchestration runs artifact work before provider apply and app work after services are running; app-step output is persisted unredacted under the user data root while only redacted `task.detail` lines cross EventService.

## Durable learnings

- Business logic stays in pure Effect; filesystem/process/network/terminal/provider/renderer effects belong behind services.
- Public contracts come from Effect Schema with tagged errors and generated schema snapshots. Machine error envelopes serialize only `_tag`, `message`, and `remediation`.
- Generated-file discipline is non-negotiable: change the generator, run codegen, include intentional outputs.
- Source and compiled CLI paths must stay behaviorally paired whenever command routing changes.
- Renderer and redaction own user-visible output; durable state uses `StateStore`; paths use `makeLandoPaths`/`PathsService`; host/provider probing uses `@lando/sdk/probe`.
- Runtime bundles for managed Linux setup need a local-engine Podman plus complete helpers (including `rootlessport`). Checksums and a ready socket do not prove networking/port publish works. Launch managed service with `CONTAINERS_CONF=.../containers.conf` — `--config <dir>` alone does not load it. Prefer `/libpod/_ping` for readiness, not `/libpod/info`.
- Log-follow lifecycle lives in the provider-neutral SDK engine (lazy finalizers, UTF-8 framing, planner-owned capability degradation). Required follow sources fail at plan time when capability is missing.
- Never classify a present-but-broken release-blocking path as an environment skip.
