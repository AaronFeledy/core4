## Codebase Patterns
- Adding typed per-call-generic methods to an existing `Context.Tag` service (e.g. `EventService.waitFor<Name>`/`query<Name>` narrowing on a `_tag` literal): the tag stays non-generic; only its method members become generic functions. Define `EventFor<Name>` as `Name extends "*" ? Union : Name extends LandoEventName ? Extract<Union,{_tag:Name}> : LooseStructural` — the loose-structural fallback for arbitrary `string` keeps every existing `subscribe(name: string)` caller compiling. When the generic methods reference a large discriminated union INLINE in the `Context.Tag<Self,{...}>` 2nd type-arg, TS throws `TS2310/TS2506` "recursively references itself as a base type"; FIX by extracting the service object into a named `export interface XShape` and passing `Context.Tag<Self, XShape>`. BUT the `sdk/src/services/index.ts` `declare class` MIRROR must stay an INLINE `{...}` type literal (NOT a reference to that interface) because `sdk-backward-compatibility.test.ts` parses `index.ts` ALONE and resolves only the inline 2nd type-arg — a cross-file interface reference reads as zero methods and fails the frozen-signature `toEqual`. Evolving a frozen tag's signatures (new methods OR a changed param shape like positional `filter?` -> `options?: {filter?,timeout?}`) requires: update the tag's entry in `sdk/test/fixtures/sdk-mvp-surface.json` (exact per-tag `toEqual`, match source spelling incl. generics), update the EXPECTED_TAGS methods list AND the `expectedMethods` array in `sdk/test/services/tags.test.ts`, and add an `## Compatibility notes` bullet in `API_COMPATIBILITY.md`. A `TaggedError` field addition (`reason?: Schema.optional(Schema.Literal("timeout"))`) is additive and needs NO schema-snapshot regen when the error is not in `JSON_SCHEMA_REGISTRY` (most `@lando/sdk/errors` aren't). For a service whose `Layer` must read a SIBLING service (e.g. `EventServiceLive` redacting its history buffer through `RedactionService`), `Effect.serviceOption(Sibling)` CANNOT see a sibling in the same `Layer.mergeAll`; provide it INTO the consumer via `Consumer.pipe(Layer.provide(SiblingLive))` in the bootstrap generator (`scripts/build-bootstrap-layers.ts` `renderMinimal`/`renderProvider`), regenerate, and reuse that redacted layer at every `Layer.provide(EventServiceLive)` site. A `cap:0` zero-allocation no-op returns a shared frozen `EMPTY_HISTORY` const (asserted via `toBe` identity). Type-level narrowing assertions use the repo's custom `Assert<Equal<A,B>>` conditional-type helpers (no `expectTypeOf` dep) and TS instantiation expressions (`ReturnType<typeof service.subscribe<Name>>`); they live in `bun test` files under `sdk/test/types/` and are proven real by a tsc break/restore (a wrong `Equal` yields `TS2344 Type 'false' does not satisfy 'true'`).
- Migrating a bespoke durable file (lock + quarantine + version-envelope) onto `StateStore` while keeping a zero-arg sync factory and unchanged public shape (the scratch registry, `core/src/scratch-app/registry.ts`): build the bucket from the dependency-free `makeStateStore()` factory (the same value `StateStoreLive = Layer.succeed(StateStore, makeStateStore())` wraps), NOT `yield* StateStore` — tests call `makeScratchRegistry()` directly and run its effects with no `StateStore` service in context. Store `A = entries-array` (NOT the full `{version,entries}` envelope) so `update` operates on the array and the json codec frames it as `{version, data:[...]}` without a redundant second version field; reconstruct the legacy envelope shape (`{version:1, entries}`) only at the service `read()` boundary. `onCorrupt:"quarantine"` + `default:[]` makes a malformed `registry.bin` (not even a valid `{version,data}` envelope) rename to `registry.bin.corrupt-<ts>` and fall back to `[]` (the StateStore `get` quarantines BOTH frame-decode and schema-decode failures). The bucket's own advisory lock (`registry.bin.lock`) now serializes mutations; the old explicit lock helper (`acquireScratchRegistryLock` on `registry.lock`) is retained as a thin delegation to a new exact-lock-path helper `acquireAdvisoryLockAt(lockPath, op)` exported from `core/src/state/lock.ts` (`withAdvisoryLock(file,...)` always derives `${file}.lock`, so it cannot lock an exact `registry.lock` path). Map `StateStoreError -> ScratchAppError` (carry the state error as `cause`, pass through `remediation`) so callers depending on `_tag`/`operation`/`message` are unchanged; add `Effect.asVoid` after `bucket.update` (it returns the new value) to keep `upsert`/`remove` returning `Effect<void>`. `ScratchRegistryLive` export is unchanged so the generated `scratch` bootstrap layer needs no regen.
- Durable on-disk state with cross-process serialization (`StateStoreLive`, `core/src/state/`) needs TWO lock layers, both load-bearing: an `O_CREAT|O_EXCL` token lockfile (`lock.ts`) for OTHER processes, AND a per-absolute-path in-process `Effect.unsafeMakeSemaphore(1)` (module-level `Map`) for same-process fibers. The file lock alone does not reliably serialize fibers under Effect's scheduler — two `acquire` retry loops interleave in the window between a holder's release-unlink and the next `O_EXCL` create, dropping a write. Never take over a lock just because its record reads back `null` (a holder mid-flush reads as null); gate the null-record takeover on the lockfile's own `mtime`. Resolve named roots inside the layer via the pure `resolveLandoRoots()`/`makeLandoPaths()` from `@lando/core/paths` (the `ManagedFileServiceLive` precedent), NOT `PathsService` — it is a sibling in the `minimal` `Layer.mergeAll`, so `Effect.serviceOption(PathsService)` cannot see it. SDK custom codecs (`encode/decode`, no version slot) own their whole file byte-for-byte and skip version framing; only built-in `json`/`binary` are versioned envelopes.
- The canonical Landofile serializer pair (`@lando/sdk/landofile` `emit.ts`/`parser.ts`) is an ASYMMETRIC round-trip contract: the emitter and parser must agree on the map-key shape. The emitter's `KEY_SHAPE` (`^[A-Za-z0-9_.-]+, all three write paths via the `entriesOf` helper), the parser's `parseMap` key match, and the parser's `parseList` list-item-map `mapMatch` regex must stay in lockstep — widening the emitter's accepted key without widening the parser's list-item matcher makes an array-of-maps key emit valid YAML that parses back as a scalar. `parseScalar` ordering is load-bearing: quoted-literal branches (`"…"`/`'…'`) MUST run before the `${` expression rejection, or quoted `${secret:…}` is wrongly rejected (the §7.8.1 secret round-trip). Strict "encoded wire form" object detection is `Object.getPrototypeOf(v) === Object.prototype || === null` (NOT `typeof object && !Array.isArray`, which silently accepts/corrupts `Date`/`RegExp`/`Map`/class instances). `emitLandofileYaml(value, { sortKeys })` opts into recursive lexicographic key ordering without reordering arrays; default is insertion order (byte-identical). Adding the option/hardening leaves the §13.2 schema snapshot unchanged (no schema registered).
- The app-plan cache key (`deriveAppPlanCacheKey`, `core/src/cache/app-plan.ts`) folds composition inputs via its `serviceInputs` slot. `AppPlanner.planApp` is two-phase for this: a cheap resolve-only pre-pass collects each service's resolved `base` + ordered FeatureRef list (base defaults first, then `resolution.features`) and the contributed app-feature DEFINITION inputs (id/priority/activatedBy/selectors/requires/pluginId/config); the cache key is derived and read from those BEFORE any plan production, so a warm hit still skips `composeService`/`legacyServicePlan`/`composeAppFeatures`. On a miss the SAME resolution objects are reused — `ServiceType.resolve()` runs exactly once per service. Hashing app-feature definition inputs is equivalent to hashing the post-activation result because activation is a pure function of the resolved drafts (which derive from featureRefs+config already in the key). A `lando`-base service type pulls its whole `LANDO_BASE_DEFAULT_FEATURE_IDS` stack through `PluginRegistry.loadServiceFeature` (deduped against resolution features, resolution wins); a planner test using `base: "lando"` must register every base-default id (no-op the unused) or use `base: "l337"` when the base stack is incidental. Compare `ServicePlan.metadata.resolvedAt` via `String(...)`, never `===` — `DateTime.Utc` objects for the same instant are reference-unequal.
- The §7.2 overlay primitive is the exported `mergeValues` in `core/src/landofile/merge.ts` (deep-merge maps, replace scalar arrays, merge object arrays by recognized identity key name/id/hostname/service, right-wins scalars). Reuse it for ANY "child overlays parent per §7.2" merge (e.g. service-type `extends:` inheritance) instead of hand-rolling a deep merge — a naive `{...a,...b}` or record-only deep merge silently drops the object-array identity-key merge the spec mandates. Service-type `type:` reference parsing must be exact-id-first (colon-bearing ids like `php:8.2`/`elasticsearch:8`/`node:22` resolve whole), only splitting at the RIGHTMOST colon into `<name>:<version>` when no exact id is registered; never first-colon-split and never fall back to version-splitting when the exact lookup fails with a cycle/depth `ServiceTypeCollisionError`. `extends:` cycle/depth (≤4 hops) detection is load-time in `composeExtendedServiceType` (the chain is static on the ServiceType); the synthesized type spreads the leaf so private bridge keys (`__legacyToServicePlan`) survive and only `resolve`/`artifacts`/`versions` are replaced. Version pinning lands via `service.image` overlay (the legacy bridge + Phase B read the authored `service`), so overlay onto the stored service, not just `normalizedConfig.image`.
- Shared SDK contract suites that prove seam usage should expose typed structural observation records (request URLs, DownloadRequest shapes, DataEndpoint tags, operation names, finalizer/probe metadata), not opaque counters; keep core-only implementation types inside `@lando/core/testing` and let `@lando/sdk/test` assert portable record shapes.

- Effect-managed "handle" objects (an App handle, etc.) that own start-state resources should: capture the ambient `Scope.Scope` via `yield* Effect.scope`, fork a per-handle `handleScope = Scope.fork(parent, ExecutionStrategy.sequential)`, and track at most one live child scope in a `Ref<Scope.CloseableScope | undefined>` (effect-v3 type is `Scope.CloseableScope`, NOT `Scope.Closeable`). Serialize lifecycle mutations with `Effect.makeSemaphore(1)`, run every ref/scope swap under `Effect.uninterruptible`, and clean up a freshly-forked scope on failed/interrupted work via `Effect.onError`. Manually closing a child scope before its parent closes is a safe no-op (Effect scopes finalize exactly once), which yields "close exactly once" + "tear down on parent close" for free.
