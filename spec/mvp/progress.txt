## Codebase Patterns
- Provider package import-boundary tests parse TypeScript source files with the TypeScript compiler API and assert static imports/re-exports avoid `@oclif/*` and `core/src/cli/*`; keep those tests under `plugins/*/test/import-boundary.test.ts`.
- Logger Live installs Effect logging with `Logger.replace(Logger.defaultLogger, logger)`; use `Logger.prettyLoggerDefault`/`Logger.none` as logger instances, while `Logger.pretty` is itself a Layer.
- FileSystem Live uses `Bun.file` for exists/stat/read/stream/delete and `Bun.write` for writes; Bun lacks a dedicated mkdir/rename API, so keep filesystem helpers constrained to Bun-native primitives and cover temp cleanup in tests.
- ShellRunner Live uses `Bun.$` with raw shell command interpolation (`${{ raw: command }}`), `.quiet().nothrow()`, and maps both parse failures and non-zero exits into `ShellExecError`.
- provider-lando setup validates the manually installed Podman CLI with `podman --version` and the daemon socket via `PodmanApiClient.info`; tests should inject a fake `podmanCommand` plus fake `podmanApi` instead of depending on host Podman.
- Compiled binary version output comes from `core/src/version.ts`, not a full `core/package.json` import; package metadata strings can leak into `bun build --compile` artifacts. `core/package.json#build:compile` runs `scripts/sanitize-compiled-binary.ts` after compile to scrub forbidden build-cache markers while keeping the binary runnable.
- Bun test discovery is now explicit in root `bunfig.toml` via `[test].include`; keep new test layers under `core/test/**/*.test.ts`, `sdk/test/**/*.test.ts`, or `plugins/*/test/**/*.test.ts` so top-level `bun test` finds them.
- `@lando/core/testing` exposes `makeTestRuntime()`/`provideTestRuntime()` for Effect-service tests; the returned runtime includes a `layer` plus mutable `calls`/`files` handles for assertions, and provider-level tests can override `RuntimeProvider` with `TestRuntimeProvider` from `@lando/sdk/test`.
- Bundled plugin codegen reads `core/build.config.ts`, writes `core/src/plugins/bundled.ts`, and should produce Biome-clean output because lint checks generated files.
- Bundled plugin codegen statically imports plugin packages and can expose package contribution maps (for example `serviceTypes`) through `BUNDLED_PLUGINS`; keep generator output idempotent and Biome-clean.
- After adding a new `plugins/*` workspace package, run `bun install` to refresh root `node_modules` workspace links before manual package-import QA from the repo root.
- Pre-OCLIF CLI fast paths must avoid static imports/re-exports of the OCLIF runner; ESM evaluates static dependencies before module bodies, so use dynamic imports and `import.meta.main` for executable entry checks.
- OCLIF init-hook tests should use a fixture CLI root and `Config.load({ root, ignoreManifest: true })` so stale `oclif.manifest.json` files do not bypass fixture commands; reset `process.exitCode` after expected OCLIF command failures so Bun exits green when assertions pass.
- Database service type implementations may extend the MVP `ServiceConfig` schema with typed convenience keys such as `database`/`port`; decode Landofile-shaped tests through `LandofileShape` so those keys are proven to survive config validation.
- For repo codegen, generate `core/oclif.manifest.json` programmatically with `Config.load({ root: CORE_ROOT, ignoreManifest: true })` instead of shelling to `bunx oclif manifest`; the CLI path can hit Bun's node_modules TypeScript stripping limitation through workspace symlinks.
- Package export-surface tests belong in `sdk/test/library/exports.test.ts` and should use dynamic `await import("@lando/sdk/...")` calls for each public subpath so Bun validates the workspace package `exports` map exactly as embedding hosts consume it.
- Provider-lando Compose emission lives in `plugins/provider-lando/src/compose.ts`; future provider lifecycle work should reuse `emitCompose(plan, { userDataRoot })`, which writes through `FileSystem.writeAtomic` under `<userDataRoot>/apps/<appId>/compose.yml`.
- RuntimeProviderRegistry Live consults `PluginRegistry` contribution buckets for installed provider ids and `ConfigService.defaultProviderId` for selection; build the service inside `Layer.effect` so methods close over dependencies and remain env-free.
- AppPlanner Live returns schema-decoded `AppPlan` values; construct the raw plan with encoded metadata date strings, then decode through `AppPlan` so branded paths/names and DateTime fields are normalized.
- BuildOrchestrator Live should close over `EventService` and `RuntimeProviderRegistry` inside `Layer.effect`, then serialize per-service provider builds with `Effect.forEach(..., { concurrency: 1, discard: true })` while publishing `pre-build`/`post-build` around each build.
- Provider-lando exec uses the Podman API exec flow (`/containers/{name}/exec`, `/exec/{id}/start`, `/exec/{id}/json`) and decodes the attach stream's 8-byte stdout/stderr frame headers; do not shell out to `podman exec`.
- Provider-lando logs should use the Podman HTTP logs endpoint (`/containers/{name}/logs`) with `stdout=true`, `stderr=true`, and `follow`/`tail` query params; decode the same 8-byte stdout/stderr multiplex frames used by exec attach.
- Provider-lando inspect should read `/containers/{name}/json`, map Podman `State.Running`/`State.Status` into the provider snapshot status/state, and return endpoints from the decoded `AppPlan` rather than mutating runtime state.
- Provider-docker mirrors provider-lando's RuntimeProvider contract flow but targets the Docker Engine HTTP API via `LANDO_TEST_DOCKER_SOCKET`/`DOCKER_HOST`; provider operations must not shell out to the `docker` CLI.

- Lifecycle event schemas in `sdk/src/events/index.ts` use `Schema.TaggedStruct("event-name", {...})`; for PRD-01 app/service/build lifecycle contracts, keep both `_tag` (from TaggedStruct) and an explicit literal `eventName` field so publishers can discriminate either way.
- Bun resolves `@lando/sdk` workspace self-imports for tests inside `sdk/test/`. Use the canonical public path (`@lando/sdk/schema`, `@lando/sdk/errors`, etc.) in SDK tests so the test exercises the same import surface plugin authors use.
- Compiled CLI fast paths must not use `Bun.file(new URL(...package.json, import.meta.url))` for package metadata; compiled binaries resolve that through Bun's virtual filesystem. Statically import `core/package.json` instead, and include it in `core/tsconfig.json`.
- Avoid `--minify` on the MVP compiled OCLIF binary; Bun 1.2.18 can produce a compiled artifact that throws during the dynamic OCLIF import.
- `tsc -b` (root `bun run typecheck`) only walks projects listed in `tsconfig.json#references`. Tests live outside those project roots, so they are not type-checked by `bun run typecheck`. Bun runs them directly. Treat a passing `bun test` plus a passing `bun run typecheck` as the test/typecheck gate; do not assume tests are TS-strict-checked.
- The SDK schema module (`sdk/src/schema/index.ts`) uses `// ====` section banners with a one-line description and a `SPEC:` reference. Keep this style when adding new schema sections so the public-API contract surface stays self-documenting.
- When relocating a contract from core to `@lando/sdk`, leave a `export { X } from "@lando/sdk/..."` re-export at the original core path so internal call sites keep working without churn. Add a short module docstring noting where the canonical home now lives.
- Pre-existing `noUncheckedIndexedAccess` errors will block any feature commit because `tsc -b` is the gate. Repair the smallest minimal fix as a separate `chore:` commit before the feature so history stays bisectable.
- Runtime Layer composition tests should build the Layer with `Effect.scoped(Layer.build(...))` and inspect supplied tags with `Context.getOption`; this verifies exact bootstrap composition without invoking unfinished service method bodies.
- Effect Schema fixtures: use `typeof MySchema.Encoded` for input/wire shapes and `typeof MySchema.Type` for decoded output. They diverge for non-trivial leaves (e.g. `Schema.DateTimeUtc` encodes as ISO-8601 string but decodes to `DateTime.Utc`). Build wire-form fixtures on `.Encoded` and produce date strings via `DateTime.formatIso(dt)`.
- SDK service tag tests can use `Context.isTag(tag)` for runtime tag identity and `Context.Tag.Service<typeof Tag>` for compile-time method shape assertions; because `sdk/test/` is outside root `tsc -b`, run targeted `tsc --noEmit` when those tests rely on type-level assertions.
- ConfigService Live reads `<userConfRoot>/config.yml`, treats the file as optional, applies `LANDO_*` environment overlays afterward, and decodes the merged object through the SDK `GlobalConfig` schema.
- For "optional in wire form, but always present after decode" fields, use `Schema.optionalWith(<Schema>, { default: () => <value> })`. The decoded type narrows the field to non-undefined, and a partial parent input (e.g. `telemetry: {}`) still yields the default. This is the one-call alternative to `Schema.optional(...) + Schema.withDefaults(...)` and pairs cleanly with nested defaults.
- `ParseResult.ArrayFormatter.formatErrorSync(err)` returns rows whose `path` is `ReadonlyArray<PropertyKey>`, not a dotted string. Match nested-field issues via `issue.path.includes("fieldName")`; chain `.includes()` for deeper paths (e.g. `path.includes("services") && path.includes("primary")`). Biome's `useLiteralKeys` flags bracket access on known properties — prefer dot-keyed services in fixtures (`services.web`, not `services["web"]`).
- When a schema is shared across SDK modules (e.g. `AppRef` referenced by both `sdk/src/schema/index.ts` and `sdk/src/events/index.ts`), define it once in `sdk/src/schema/` and `import` + `export type` re-export it from the consuming barrel. The SDK's barrel pattern keeps the canonical home in `@lando/sdk/schema` while preserving the public surface from the consuming module.
- Contract-locking tests for a `Schema.Struct` can introspect the AST instead of relying on snapshots: `MyStruct.fields` is the field map by name; `Schema.Literal(...).literals` returns the literal options array; `Schema.Boolean.ast._tag === "BooleanKeyword"`; `Schema.Array(...).ast._tag === "TupleType"`. Use these to assert "exact field set + exact literal options + correct kind" so any drift fails closed.
- To verify a struct has no accidentally-optional fields, iterate the expected field name set and omit each one in turn — a single-omission test misses optional-field accidents that a per-field loop catches.
- To probe an SDK schema's internals interactively, drop a throwaway `*.tmp.test.ts` under `sdk/test/schema/` (Bun resolves the workspace `@lando/sdk` self-import inside the workspace tree). Scripts run from `/tmp` or repo root cannot resolve workspace exports, so the temporary-test approach is the path of least resistance.
- Biome's lint rules collapse short multi-line array spreads onto a single line; keep contract-test field-set initializers single-line where reasonable to avoid an unnecessary lint round-trip.
- Bun's `Error` superclass auto-populates `line` and `column` properties with the constructor's source location. `Schema.TaggedError` extends `Error`, so a TaggedError field declared as `Schema.optional(Schema.Number)` and named `line` or `column` will silently report Bun's source-line number when the caller omits the field (the optional declaration leaves the property unset, so the Bun built-in shines through). Use `Schema.UndefinedOr(Schema.Number)` instead — Effect explicitly assigns `undefined` to the instance property, overriding the Bun built-in. The same trap applies to any TaggedError field whose name collides with an `Error` built-in. Verify suspect cases with a throwaway probe: omit the field, log `instance.fieldName` and `Object.hasOwn(instance, "fieldName")` — `Schema.optional` shows the Bun bleed-through, `Schema.UndefinedOr` shows `undefined` with `hasOwn === true`.
- EventService Live is `PubSub.unbounded`-backed; use `Stream.fromPubSub(pubsub)` for scoped subscriptions, support exact `_tag` filtering plus `"*"` for all lifecycle events, and start subscriber fibers before publishing in tests because PubSub does not replay by default.
- CacheService Live is in-memory only for MVP; store unknown values, decode them on read with the caller's Effect Schema, and use Effect `Clock` for TTL so tests can run under `TestContext.TestContext` with `TestClock.adjust(...)`.
- Provider plugin capability tests should inject a tiny Podman API client into the provider layer so unit tests do not need a real daemon; keep real HTTP-over-UNIX request construction in a separate helper and reserve `LANDO_TEST_PODMAN_SOCKET` for socket-backed QA.
- Provider-lando lifecycle code should use the `PodmanApiClient.request` HTTP-over-UNIX seam for Podman API calls; fake it in unit tests and reserve live Node/Postgres checks for `LANDO_TEST_PODMAN_SOCKET`.
- Provider-lando stop/remove lifecycle should use Podman API container stop + forced container delete and per-app network delete; `app:stop` preserves app-scoped stores by never calling the Podman volumes delete API.
- The SDK provider contract suite executes `apply` -> `inspect` -> `destroy` against a fixture whose provider id matches the candidate provider; RuntimeProvider implementations that cache applied plans must wire `destroy` through provider teardown and clear that cache.
