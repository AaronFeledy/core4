## Codebase Patterns

- Docs component prop schemas that participate in the schema gate need exports from `@lando/sdk/docs/components`, `@lando/core/docs/components`, and `@lando/sdk/schema`, plus `scripts/build-schema-snapshot.ts` / `sdk/API_COMPATIBILITY.md` updates.
- New public `@lando/sdk/schema` exports must be listed in `sdk/API_COMPATIBILITY.md`; the SDK backward-compatibility test compares additive exports against that document.
- Extending SDK service tag methods requires updating all in-tree service doubles plus `sdk/test/services/tags.test.ts` and `sdk/test/fixtures/sdk-mvp-surface.json`.
- `build-guide-scenarios --only <guideId>` is for author-command regeneration and must clear only `test/scenarios/generated/guides/<guideId>/`; full codegen without `--only` remains responsible for clearing the whole generated guide root. This avoids parallel test/codegen races with unrelated generated guides.
- The scenario source mapper is a pure output post-processor (`scripts/test-reporters/scenario-source-mapper.ts`) with fixture-pair tests under `core/test/scenarios/reporter/`; `docs:scenario` applies it to captured `bun test` stdout/stderr for generated guide paths.
- Source-mapper fallback behavior stays header/context driven: preserve the generated frame, emit a `GuideSourceMapWarning` frame when `@source` / `@scenario` / `guideId` is missing, and fall back to whole-guide `bun run docs:scenario <guideId>` instead of deriving ids from generated paths.
- Generated guide scenarios may author reader-facing commands as `lando ...`; `ScenarioContext.runCli` strips the leading binary token for the in-process runner and advances its working directory after `init` / `UseFixture` so later `lando start` / `lando destroy` calls run inside the scaffolded app or copied fixture. NOTE: `start` and `destroy` are currently short-circuited by an in-context fake (`runScenarioLayerCommand` in `core/src/testing/scenario-context.ts`) — `cli.runCli` does not yet accept a runtime/layer override, so the scenario's `runtime` field built from `makeTestRuntime` is not wired into the CLI. Real `TestRuntimeProvider` injection through `cli.runCli` is follow-up work; until then the invalid-Landofile regression coverage is by string-match against the fixture, not by exercising the real Landofile validator.

- Scenario transcripts are persisted by `withScenarioContext` on `Effect.onExit` under `dist/transcripts/guides/<guideId>/<scenarioId>.json`; keep top-level `startedAt` / `finishedAt` ISO-valid and apply timestamp masking only to stdout/stderr-style transcript fields.
