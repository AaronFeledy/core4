Important: when editing files in `spec/` with the `Edit` tool, do **not** put a literal `$` in `new_string` — it gets parsed as a JS `String.prototype.replace` pattern (`$$` → `$`, `$&` → match, `` $` `` → entire pre-match file content, etc.) and silently corrupts the file. Instead, double every `$` in `new_string` to `$$` (so `${VAR}` → `$${VAR}`, `` Bun.$`echo hi` `` → `` Bun.$$`echo hi` ``), or use `Write` for the whole file. `old_string` is matched literally and needs no escaping.
