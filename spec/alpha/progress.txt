- Live integration tests stay gated on explicit env vars and host runtime prerequisites.

## Codebase Patterns

- Planner capability gates live in `core/src/services/planner.ts`; service-plan fields that require provider support should fail before apply with `CapabilityError` carrying service, feature, capability, provider id, and remediation.
- `meta:setup` runs at provider bootstrap through `RuntimeProviderRegistry.select()`; compiled `$bunfs` commands that bypass OCLIF need matching handlers in `core/src/cli/run.ts`.
- `provider-lando` setup state is persisted under `<userDataRoot>/providers/provider-lando/setup-state.json` when a state directory is configured.
