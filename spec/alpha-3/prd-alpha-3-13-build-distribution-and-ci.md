# PRD: ALPHA3-13 — Build, distribution & CI matrix

## Introduction

Alpha 3 closes the build/distribute/CI surface needed for `4.0.0-alpha.N` to ship on the `dev` channel. The compile pipeline grows to all five platform binaries (`darwin-x64`, `darwin-arm64`, `linux-x64`, `linux-arm64`, `windows-x64`), the AOT bootstrap-layer codegen (§17.2) lands, asset embedding (recipes, schemas, OCLIF manifest) passes the import-boundary test, the per-PR CI matrix runs every §13.1 test layer on every platform, the nightly e2e job runs against `@lando/provider-lando`, the weekly provider matrix exercises all blessed engines, and `@lando/core` `4.0.0-alpha.N` publishes to npm on the `dev` tag.

Depends on: **ALPHA3-01 through ALPHA3-12** (this is the integration / shipping PRD).

## Goals

- Build all five platform binaries reproducibly via `bun build --compile`.
- Land the AOT bootstrap-layer codegen so runtime `Layer.mergeAll` chains disappear.
- Land asset embedding for recipes, schemas, and the OCLIF manifest with no runtime FS read.
- Run the §13.6 per-PR matrix on all five platforms and add nightly + weekly cadence jobs.
- Publish `@lando/core` `4.0.0-alpha.N` on the `dev` npm tag.

## User Stories

### US-183: build `darwin-arm64` binary

**Description:** As CI, the `build-darwin-arm64` job produces `dist/lando-darwin-arm64` via `bun build --compile` and runs smoke + `--help` checks before uploading the artifact.

**Acceptance Criteria:**
- [ ] Runner is `macos-14` (or current Apple-silicon image); Bun version pinned per `.bun-version`.
- [ ] Build artifact uploaded with 14-day retention.
- [ ] Smoke covers `--version`, `--help`, and `lando shellenv`.
- [ ] Tests pass; typecheck passes; lint passes.

### US-184: build `darwin-x64` binary

**Description:** As CI, the `build-darwin-x64` job mirrors US-183 for Intel macOS.

**Acceptance Criteria:**

- [ ] Build artifact uploaded with 14-day retention.
- [ ] Tests pass; typecheck passes; lint passes.

### US-187: AOT bootstrap-layer codegen (§17.2)

**Description:** As a maintainer, the runtime no longer uses runtime `Layer.mergeAll`/`Layer.provide` chains outside the §17.2 codegen output; bootstrap layers are emitted ahead of time.

**Acceptance Criteria:**
- [ ] Codegen script (`scripts/build-bootstrap-layers.ts`) emits per-bootstrap-level Live Layers under `core/src/runtime/generated/`.
- [ ] `core/src/runtime/layer.ts` consumes the generated module; no runtime `Layer.mergeAll` outside the codegen output.
- [ ] AGENTS.md "AOT bootstrap-layer codegen not yet shipped" interim bullet removed.
- [ ] Tests pass; typecheck passes; lint passes.

### US-188: asset embedding for recipes / schemas / OCLIF manifest

**Description:** As the compiled binary, bundled recipes, schemas, and the OCLIF manifest are embedded at build time — no runtime FS read of these assets.

**Acceptance Criteria:**
- [ ] Build pipeline reads recipes / schemas / OCLIF manifest at compile time and emits them into the binary's data section.
- [ ] Import-boundary test asserts no runtime FS read of `recipes/`, `sdk/schema/`, or `oclif.manifest.json` from the compiled binary path.
- [ ] `bundled.ts` is populated by codegen (`scripts/build-bundled-plugins.ts`); hand-edits forbidden.
- [ ] Tests pass; typecheck passes; lint passes.

**Description:** As CI, every PR runs `static-checks`, `build-<platform>`, and `provider-integration-<platform>` (where supported) on all five platforms.

**Acceptance Criteria:**
- [ ] `.github/workflows/ci.yml` (regenerated by codegen) covers the 5-platform matrix.
- [ ] All §13.1 test layers (unit, Effect service, CLI, scenario, recipe, library, provider contract) run green on every platform.
- [ ] Compile time + test wall-clock are tracked; per-platform timeout caps documented.
- [ ] Tests pass; typecheck passes; lint passes.

### US-190: nightly e2e against `@lando/provider-lando`

### US-191: distribution rehearsal in nightly job

**Description:** As CI, the nightly job rehearses the full distribution flow (build, package, upload artifact set) without actually publishing to GitHub Releases.

**Acceptance Criteria:**
- [ ] Rehearsal produces a tagged artifact bundle (binaries + sha256 sums) under workflow-artifact storage.

### US-192: weekly provider matrix

**Description:** As CI, a weekly cron job runs the provider contract suite against Docker Desktop, Docker Engine, Podman Desktop, Podman, Lima, and OrbStack on the platforms each supports.

**Acceptance Criteria:**
- [ ] `.github/workflows/provider-matrix.yml` declares the weekly cron; matrix cells skip where the engine is not installable in CI.

### US-193: publish `@lando/core` `4.0.0-alpha.N` on the `dev` npm tag

**Description:** As a publisher, the release pipeline tags `4.0.0-alpha.N` and publishes `@lando/core` (and the bundled plugin packages) on the npm `dev` tag.

**Acceptance Criteria:**
- [ ] `@lando/sdk/test` is in `package.json#exports`; the contract suite runner has documented args.

- [ ] Tests pass; typecheck passes; lint passes.

## Functional Requirements

- FR-1: All five platform binaries build via `bun build --compile` from `core/bin/lando.ts` per the §17.1 stage rules.
- FR-2: AOT bootstrap-layer codegen lands; runtime `Layer.mergeAll` outside generated code is forbidden.
- FR-3: Asset embedding for recipes / schemas / OCLIF manifest passes the import-boundary test.
- FR-4: Per-PR matrix runs all §13.1 layers on all 5 platforms; nightly e2e + distribution rehearsal run on Linux x64; weekly provider matrix runs on supported cells.
- FR-5: `@lando/core` `4.0.0-alpha.N` publishes to npm on the `dev` tag; SDK contract tests publish alongside.
- FR-6: `.github/workflows/*.yml` is generated by `scripts/build-ci-workflow.ts` and friends; hand-edits forbidden (extends the MVP convention).

## Non-Goals

- Code signing / notarization (Beta 1).
- GPG-signed `SHA256SUMS` + cosign signatures (Beta 1).
- CycloneDX SBOM + SLSA v1.0 provenance attestation (Beta 1).
- Self-update + update manifest at `https://update.lando.dev/v4/dev.json` (Beta 1).
- Curl-pipe installer scripts (`get.lando.dev/install.{sh,ps1}`) (Beta 1).
- Homebrew / scoop / winget packaging (Phase 6).
- Schema artifact CDN (`schemas.lando.dev/v4/`) (GA).

## Technical Considerations

- The §17.1 stage-7 rule (compiled binary derives install dir from `process.execPath`, not `import.meta.url`) is already enforced; extend the same rule to any new compiled-binary fast paths.
- Multi-platform CI compile time will grow significantly — track wall clock and prune cache aggressively per `.github/workflows/ci.yml` (regenerated).
- Asset embedding strategy: prefer `Bun.embeddedFiles` over runtime FS reads; the import-boundary test must reject any `node:fs` import inside the binary that touches bundled-asset paths.
- The nightly job's distribution rehearsal is a dry run — it generates artifacts but does not push them; the Beta 1 release pipeline picks up from there.

## Success Metrics

- All five binaries build green per-PR; total CI wall clock under the documented per-platform cap.
- Nightly e2e + distribution rehearsal stay green for at least two consecutive weeks before tagging `4.0.0-alpha.1`.
- Weekly provider matrix is green on every supported engine.
- `@lando/core` `4.0.0-alpha.N` installs from `npm install --tag next @lando/core` on every Alpha 3 platform.
